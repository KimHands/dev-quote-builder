# AI 요구사항 정리기 (mindlogic) Implementation Plan (Plan 4/6)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox steps.

**Goal:** 비개발자가 "당근마켓 같은 거" 같은 자유 입력을 넣으면 AI가 구조화된 기능 명세 + 추천 기능모듈을 스트리밍으로 돌려주는 정리기. **로그인 게이트(가입 유도) + 다층 rate limit + 입력 cap**. 게이트 앞엔 "값 미리보기"(ADR 0005 보정).

**Architecture:** `/api/ai-parse`(POST) = 로그인확인(`auth()`)→rate limit(AiParse DB 카운트 + env 한도 + 전역 킬스위치)→입력 cap→mindlogic 스트리밍 호출→토큰 스트림 응답 + AiParse 기록. mindlogic은 **OpenAI 호환**(라이브 검증 완료): `openai` SDK, baseURL=`MINDLOGIC_BASE_URL`, `POST /chat/completions`, Bearer 인증, model `claude-sonnet-4-6`, `stream:true`. UI는 터미널 블록 스트리밍 + 비로그인 시 값-미리보기+카카오 로그인 카드.

**Tech Stack:** Next.js 16 · openai(npm) SDK · Prisma 7 · next-auth v5(`auth()`) · Vitest

**확정 API 사실(라이브 프로브로 검증):**
- base `https://factchat-cloud.mindlogic.ai/v1/gateway` + `/chat/completions` → HTTP 200
- 응답 표준 OpenAI shape(`choices[0].message.content`, 스트림 시 delta)
- 인증 `Authorization: Bearer ${MINDLOGIC_API_KEY}`, model `claude-sonnet-4-6`

**rate limit env(.env, SET):** USER_DAY=10 · USER_MIN=3 · IP_DAY=20 · INPUT_CHAR_CAP=1000 · OUTPUT_MAX_TOKENS=1500 · GLOBAL_KILL_DAY=200

**참조:** [ADR 0005](../../adr/0005-kakao-auth-ai-gate.md) · CONTEXT.md §AI rate limit·§엔티티 AiParse

## Task 3: rate-limit 로직 (TDD)
- [ ] **Step1 failing test** `tests/lib/ai/ratelimit.test.ts`: 순수 결정 함수 `decideLimit({userDay,userMin,ipDay,globalDay}, limits)` → `{allowed:boolean, reason?:string}`:
```ts
import { it, expect } from "vitest";
import { decideLimit } from "@/lib/ai/ratelimit";
const L = { userDay:10, userMin:3, ipDay:20, globalDay:200 };
it("한도 내면 허용", () => {
  expect(decideLimit({ userDay:2, userMin:1, ipDay:5, globalDay:50 }, L)).toEqual({ allowed:true });
});
it("전역 킬스위치 우선", () => {
  expect(decideLimit({ userDay:0, userMin:0, ipDay:0, globalDay:200 }, L).allowed).toBe(false);
});
it("사용자 일일 초과", () => {
  expect(decideLimit({ userDay:10, userMin:0, ipDay:0, globalDay:0 }, L).allowed).toBe(false);
});
it("분당 초과", () => {
  expect(decideLimit({ userDay:0, userMin:3, ipDay:0, globalDay:0 }, L).allowed).toBe(false);
});
it("IP 일일 초과", () => {
  expect(decideLimit({ userDay:0, userMin:0, ipDay:20, globalDay:0 }, L).allowed).toBe(false);
});
```
- [ ] **Step2** RED. **Step3** 구현: 전역(globalDay>=limit) 먼저, 그다음 userDay/userMin/ipDay 각 `>=` 초과 시 차단, 아니면 allowed. + `readEnvLimits()`(env에서 숫자 파싱, 기본값 10/3/20/200). + `countWindows(prisma, userId, ip)`(AiParse에서 오늘/최근1분/IP오늘/전역오늘 카운트) 헬퍼(이건 DB라 테스트는 decideLimit 순수부만).
- [ ] **Step4** GREEN. 커밋 `feat: AI rate-limit decision logic (TDD)`

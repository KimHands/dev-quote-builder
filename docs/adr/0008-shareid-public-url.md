# ADR 0008 — 공개 견적 URL은 추측불가 shareId (순차 PK 노출 금지)

- **상태:** Accepted · **날짜:** 2026-05-30 (grill-with-docs)

## 결정
저장된 Quote의 공개 공유 링크 `/q/{...}`는 내부 PK(`id`)가 아니라 별도의
**추측불가 랜덤 식별자 `shareId`(nanoid)**를 사용한다. `id`는 내부 전용으로 외부에 노출하지 않는다.

## 근거
- Quote(=Lead)에는 **연락처(이름·전화·카톡ID)**가 담긴다.
- PK가 순차/추측 가능하면(`/q/1`, `/q/2`...) 누구나 숫자를 증분해 **타인 리드를 열람**할 수 있다(IDOR).
- shareId를 랜덤으로 분리하면 링크 소지자만 열람, 인접 레코드는 노출되지 않는다.

## 대안
- **PK 직접 노출:** 구현 단순하나 IDOR 취약. 기각.
- **클라이언트사이드 인코딩 공유(백엔드 없음):** 이전 grill-me 메모. 풀스코프/백엔드 도입(ADR 0001)으로 폐기됨 — 영속·재현·admin 추적 불가.

## 불변식
공개되는 식별자와 내부 PK는 항상 분리. `/q/[shareId]` 라우트는 shareId로만 조회.
관련: [[ADR 0001]](풀스코프·DB 기반 공유링크), CONTEXT.md §엔티티 Quote.

## 되돌림 조건
거의 없음(보안 기본값). 만약 단축 URL·QR 등 추가 시에도 shareId 위에 얹는다.

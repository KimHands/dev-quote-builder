# ADR 0001 — 풀스코프 선구축

- **상태:** Accepted (논쟁적 — 세 신호가 반대했으나 유저가 유지)
- **날짜:** 2026-05-30

## 맥락
office-hours 결론은 "진짜 병목은 견적 자동화가 아니라 신뢰+아웃리치, 신뢰페이지+문의폼만 먼저
띄우고 영업하라"였다. 운영자는 이를 폐기하고 견적저장+알림+공유링크+관리자+AI정리기+이메일을
**처음부터 전부** 개인 서버에 구축하기로 결정.

## 결정
명세서 Phase 1~6 풀스코프를 선구축한다. 단, 정밀 man-day 엔진만 C단계로 연기(→ ADR 0007).

## 근거
- 포트폴리오·학습 가치("제대로 풀스택 한 번").
- 기존 lab-server 인프라(Docker/Tunnel/Loki/Telegram)가 이미 깔려 운영 부담이 추정보다 낮음.

## 반대 의견 (기록)
office-hours + eng-review Step0 복잡도 게이트 + Codex 외부챌린지 **세 신호 모두** "첫 수주엔
과하다, 미니 SaaS가 아니라 신뢰페이지+영업이 먼저"라고 일치. 유저가 알고도 유지 결정.

## 되돌림 조건
한 달 내 첫 상담/계약 0건이고 빌드가 발목을 잡으면 → 신뢰페이지로 축소하고 아웃리치 집중.

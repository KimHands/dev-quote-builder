# ADR 0005 — 카카오 OAuth + AI 정리기 로그인 게이트

- **상태:** Accepted · **날짜:** 2026-05-30

## 결정
- 인증은 Auth.js(NextAuth) + **카카오 OAuth**. 엔드유저 + admin role 플래그(운영자만).
- **AI 요구사항 정리기를 로그인 뒤에만 노출** → 회원가입 유도 수단으로 사용.
- AI는 교내 mindlogic 게이트웨이(`factchat-cloud.mindlogic.ai/v1/gateway[/claude]`,
  OpenAI/Anthropic SDK 호환, x-api-key, 서버사이드 전용). 응답 스트리밍.
- 다층 방어: IP rate limit + 일일 상한 + 입력 길이 cap + 서버키 + Cloudflare WAF/Turnstile.

## 근거
- 카카오 = 한국 비개발자 원탭, 가입 마찰 최소(가입 유도 목표와 정합).
- AI 키 쿼터(운영자 부담)를 로그인+rate limit으로 보호.

## 리스크/주의
- 기본 견적 플로우는 **공개**(저마찰), AI만 게이트 — 견적 자체 전환율을 떨어뜨리지 않도록.

## grill-with-docs 재검증 (2026-05-30) — 유지 + 보정 2개
도전: 주 리드 퍼널(견적 제출)은 비로그인 허용인데 AI만 로그인 필수 = "값 보기 전 가입" 누수 위험.
결론: **유지**(주 퍼널을 막지 않으므로 게이트 정당) + 아래 2개 보정으로 *가정→검증* 전환.
1. **비대칭을 의도적 설계로 명문화:** 견적 제출=비로그인(폼에서 연락처 확보) / AI 정리기=로그인(비싼 쿼터 보호 + 보너스). 의도된 비대칭.
2. **로그인 벽 앞 "값 미리보기":** 게이트 카드에 *실제 예시 출력*(예: "당근 같은 거"→회원/채팅/위치/결제 자동체크 모습)을 정적으로 보여준 뒤 "카카오 1탭" CTA. value-first와 화해(가입 후회 방지).

검토·기각한 대안: AI 익명 1~2회 허용(Turnstile+IP) — 쿼터 노출↑·구현 복잡, 카카오 원탭이면 이득 작음.

## 되돌림 조건
보정에도 로그인 게이트가 AI 사용·가입을 오히려 죽이면 → 게스트 1회 체험 후 가입 유도로 완화.

# ADR 0007 — 티어 분류 룩업 (정밀 man-day 엔진은 C단계)

- **상태:** Accepted · **날짜:** 2026-05-30

## 결정
"예상 예산대"는 정적 **티어 분류 룩업**(if/else로 선택값 → 3 패키지 티어 → 구간)으로 구현.
명세서 §6의 정밀 man-day 엔진(effort 합산 × multiplier 5종 × 버퍼)은 **C단계로 연기**.

## 근거
- 엔진의 man-day 상수는 운영자가 아직 갖지 못한 데이터(실제 작업속도) 위에 서 있음.
- 실제 1~2건 수주 후 §6.3 상수를 역산·보정해야 엔진이 정확해짐. 순서: 검증 → 엔진.
- 클라에는 확정가가 아니라 "예산대 + 상담 후 확정"만 약속 → 룩업으로 충분.

## 불변식
가격/티어 로직은 `lib/quote/` 단일 모듈을 클라+서버 공유. 서버가 제출 시 재계산해 저장
(클라 금액 불신). 엔진 교체 시 이 모듈만 바꾸면 양쪽 반영.

## 되돌림/진화 조건
유료 계약 1~2건 완료 + 실제 man-day 기록 확보 → §6 정밀 엔진 구현(TODOS.md). "내 견적
사이트로 내 외주를 받는다" 스토리 완성.

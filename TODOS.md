## C단계: 정밀 man-day 견적 엔진
- **What:** 첫 1~2건 수주 완료 후, 실제 작업속도로 명세서 §6.3 man-day 상수를 역산·보정하고,
  티어 분류 룩업을 정밀 엔진(multiplier·버퍼)으로 교체.
- **Why:** 현재 티어 룩업은 검증 안 된 추정값. 실제 작업 데이터가 있어야 정확해짐.
- **Pros:** 정확도↑ + "내가 만든 견적 사이트로 내 외주를 받는다" 스토리 완성(포트폴리오 가치).
- **Cons:** 실수주 전엔 착수 불가.
- **Depends on:** 유료 계약 1~2건 완료 + 실제 공수(man-day) 기록.
- **Context:** office-hours/grill에서 "엔진은 검증의 보상"으로 합의. lib/quote/ 모듈만 교체하면
  되도록 설계됨(클라+서버 공유 구조라 한 곳만 바꾸면 양쪽 반영).

## 프리랜서 플랫폼(크몽) 등록
- **What:** 네트워크 수주로 리뷰 1~2개 확보 후, 크몽에 3패키지 상품 등록(에스크로 안전결제 활용).
- **Why:** 리뷰 0 상태 등록은 노출·전환 바닥. 실적 생긴 뒤가 효율적.
- **Pros:** 보조 유입 채널 + 낯선 발주자 떼임 방지(에스크로).
- **Cons:** 저가 경쟁·플랫폼 수수료.
- **Depends on:** 네트워크 수주 1~2건 + 리뷰 확보.
- **Context:** 위시켓(입찰형)보다 크몽(패키지 상품형)이 3패키지 구조와 맞음.
